Installation

Install CNOCTL

The CLI can be installed with a curl script, brew or by downloading the binary from the releases page. Once installed you'll get the cnoctl command.

Install with curl:

@@include('./html-templates/documentation/_token.html',{ comand:'$ curl -sSL https://raw.githubusercontent.com/beopencloud/cno/main/scripts/cnoctl.sh | sh' })

Install with brew on MacOs:

@@include('./html-templates/documentation/_token.html',{ comand:'$ brew tap beopencloud/cno' }) @@include('./html-templates/documentation/_token.html',{ comand:'$ brew install cnoctl' })

Install from release:

In the cnoctl release page (https://github.com/beopencloud/cno/releases) download, unarchive the version needed and move the binary to your system PATH.

Example for version v0.0.1 on MacOs:

@@include('./html-templates/documentation/_token.html',{ comand:'$ wget https://github.com/beopencloud/cno/releases/download/v0.0.1/cnoctl_0.0.1_Darwin_x86_64.tar.gz' })

Dataplane

The data plane has :

  • The CNO Agent installed in each kubernetes Cluster
  • The CNO Onboarding Operator
  • The CNO CD Operator

Command Line Interface

CNO CLI(cnoctl)

The CNO command-line tool, cnoctl, allows users to perform actions on the CNO platform.

  • CNO components
  • Control Plane
    • CNO API
    • CNO UI
    • CNO Database
    • CNO IAM
    • CNO Broker
    • CNO Notification
  • Dataplane
    • CNO Onboarding Operator
    • CNO CD Operator
    • CNO Agent
  • Command Line Interface
    • CNO CLI(cnoctl)
