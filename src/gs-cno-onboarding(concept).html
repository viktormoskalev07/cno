CNO - Onboarding

The introduction of new technology is not easy in large companies. Usually it is met with a lot of resistance from employees. Or technology is adopted but the whole organization is not onboarded well enough to really take advantage of it. CNO helps you to build the best adoption program that minimizes resistance and aligns cloud-native technology with Business.

Project and Teams Onboarding

With this onboarding feature, it will be much easier for the IT Teams to onboard organizational teams and projects. From CNO UI, each product owner in the company can now create a logical project by specifying exactly what the project needs in term of:

  • Resources Requests, like CPU, Memory or Storage
  • Environments across multi-cloud Kubernetes Clusters.
  • The developers working on the project
  • The administrators who are maintaining those applications day-to-day

CNO environments or Kubernetes Multi-cluster namespaces

In one project, it will be possible to specify many environnements, e.g. Dev, Staging, Production or any environment you want. For example, in each project, it is possible to create a Dev environment on AWS EKS, Staging environment on OpenShift and a Production environment on Kubernetes Vanilla located in your own datacenter.

Teams

After the onboarding phase, all team members in the project will have everything needed to start working. Administrators can centralize the management of Kubernetes clusters and developers can set up their environment and deploy their applications without knowing any cloud providers or any Kubernetes platform.
